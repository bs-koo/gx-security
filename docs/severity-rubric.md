# 심각도 판정 루브릭 (Severity Rubric)

gx-security의 확정 취약점 심각도를 스킬마다 들쭉날쭉하지 않게 판정하기 위한 공통 기준.
각 `detecting-*` / `exploiting-*` 스킬의 리포트는 이 루브릭을 따른다.

## 판정 축

심각도 = **악용 가능성**(얼마나 쉽게 실제로 뚫리나) × **영향도**(뚫리면 피해가 얼마나 큰가).

| 축 | 낮음 | 중간 | 높음 |
|---|---|---|---|
| **악용 가능성** | 이론적·특수 조건 필요 | 조건부(인증 필요·특정 입력) | 인증 불요·자동 확정·공개 노출 |
| **영향도** | 정보성·간접 | 부분 노출·제한적 데이터 | 인증우회·전체 데이터 유출·RCE |

## 등급 정의

| 등급 | 기준 | 예시 |
|---|---|---|
| **Critical** | 악용 쉬움 + 영향 치명적. 즉시 조치(24시간 내) | 인증 우회로 관리자 API 전체 노출(BFLA), 원격 코드 실행(웹쉘 업로드+실행 확인), JWT 서명키 유출로 토큰 위조 |
| **High** | 악용 가능 + 민감 피해. 우선 조치 | 동적 확정된 SQLi(데이터 추출), 저장형 XSS(세션 탈취), 평문 시크릿(형상관리 유출 경로), 임의 파일 읽기(경로조작 확정) |
| **Medium** | 조건부 악용 또는 부분 영향 | 위험 확장자 업로드 수용되나 웹루트 저장 미확인, Secure 미설정 쿠키, 미검증 리다이렉트(피싱 경유), 인증 필요한 IDOR 후보 |
| **Low** | 이론적·정보성. 심층방어 권고 | 테스트 스코프 더미 시크릿, CSP 부재(다른 XSS 방어 존재), 스택트레이스 노출 등 정보성 |

## 적용 규칙

1. **동적 미확정은 등급을 올리지 않는다.** 정적 후보만 있고 동적 확정(실제 발사)이 없으면 "정적 추정(미확정)"으로 표기하고 High 이상으로 단정하지 않는다(특히 IDOR/BFLA·SSRF).
2. **완화 요인을 반영한다.** 유출 경로가 끊겨 있으면(예: 평문 시크릿이 `.gitignore`로 형상관리 미추적) 한 단계 낮춘다.
3. **근본원인 수렴을 우선한다.** 단일 설정 결함이 다수 후보를 동시 유발하면, 파생 항목을 각각 세지 말고 근본원인 1건으로 묶어 그 심각도로 판정한다.
4. **grep-fallback 저신뢰를 명시한다.** 엔진이 grep-fallback이면 미탐 위험을 리포트에 경고로 남긴다(등급 자체는 확정 근거로 판정).
